# CS611 Assignment 2 — ML Pipeline for Loan Default Prediction

## What this project is
An end-to-end, production-style ML pipeline that builds on an Assignment 1 medallion
data pipeline (bronze → silver → gold) and adds model training, inference, and monitoring,
all orchestrated by Airflow running in Docker.

Goal: predict whether a customer will default on a cash loan at application time.
Label definition: `dpd >= 30` at `mob == 6` (30 days past due, 6th month on book).

## What the four deliverables of Assignment 2 are
1. Train models, evaluate, select best, store artifact in a model bank (a folder of `.pkl`s).
2. Retrieve the best model, run inference across a time period, store predictions as a gold table.
3. Monitor model performance AND stability across time, store results as a gold table.
4. Visualise performance + stability over time (for the slide deck / Task 2).
Plus: Airflow DAG that runs the whole thing and can be backfilled; runs via docker-compose.

## Model artifact format (KEEP CONSISTENT across train / inference / monitoring)
The `.pkl` is a dict with these keys:
- `model`                      : trained XGBClassifier (currently 20 clickstream features fe_1..fe_20)
- `model_version`              : str, e.g. "credit_model_2024_09_01"
- `preprocessing_transformers` : {"stdscaler": fitted StandardScaler}
- `data_dates`                 : train/test/oot window dates + ratio
- `data_stats`                 : row counts + bad rates per split
- `results`                    : auc/gini for train/test/oot
- `hp_params`                  : XGBoost hyperparameters
